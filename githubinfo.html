All About Github

Q.What is a version control system?

A. A version control system (VCS) is a system that records and saves changes to a file or multiple files, and their version histories.


Q. What is Git?

A. A free and open source VCS used for small and large projects.


Q.What is Github?

A.Github is a code hosting platform that uses Git. It allows people to work on projects both together and individually from anywhere.


What is the difference between Git and Github?

A.Git is a tool used to manage multiple versions of source code edits that are put into files in a Git repository. Github is a location to upload Git repositories.


Who started Github and how was it started?

A. Github was created by Chris Wanstrath, P.J. Hyett, Tom Preston-Werner, and Scott Chacon.They used Git and wrote the rest of the interface on Ruby on Rails and Erlang.

Git/Github terms:

  • Repository
      • A repository is an area with all your project files and their version histories.There are two types: local and remote. A local repository is on an idividual computer for one person to access. Remote repositories are hosted on servers for multiple people to access.
  • Commit
      • An individual change to a file.
  • Fork
      • A copy of a repository.
  • Push
      • Sending an edit in a local repository to the remote repository.
  • Pull Requests
      • Also called a merge request.It is a request to merge edits or commits, into a project. It gives team members a chance to propose, suggest, or collaborate on edits to a code or project. The request usually occurs in a branch to protect the original code.
  • Workflows
      • Automated processes that you set up in your Github repository. There are three main types: sequential, state machine, and rules-driven.
  • Issues
      • A section under a repository. Under the main page of a repository you can select Issues to report, discuss, and fix an issue that is found. It also allows you/your team to track progress over time.
  • Raw Button
      • A button that opens the file in raw form (without any HTML formatting).
  • Blame Button
      • A button that can be used to examine any code line. It allows you to see when it was last edited, who edited it, and what was done.
