Operating Systems Terminology

Subject Subcategories
Network Classifications
  • LAN:"Local Area Network". A collection of devices connected together in one physical location. The two primary ways to enable LAN connections are ethernet and wi-fi.
  • MAN:"Metropolitan Area Network". A computer network that connects computers witin a metropolitan area. It is larger than a LAN but smaller than a WAN.
  • WAN: "Wide-are Network". A computer network that connects smaller networks over a large distance.
Network Models
  • Bus Network:A LAN type in which each node (device) is connected to a main cable or link called a bus.
  • Star Network: AKA a hub-and-spoke network. A LAN type in which all nodes connect directly back to a singular computer (a hub).
Protocols
  • CSMA/CD Ethernet:"Carrier Sense, Multiple Access with Collision Detection". A networked host listens to the bus and waits until there's no traffic then sends a packet. If there is a collision on the bus, the host will wait a random amount of time and then attempt to resend until the packet is sent.
  • CSMA/CA Wifi:"Carrier Sense, Multiple Access with Collision Avoidance". On wireless networks, a host will wait until there is a channel that has been silent for a certain amount of time then transmits. If a collision happens, the host will wait a short period of time then attempt to retransmit.
Combining Networks
  • Repeater: A device that passes signals back and forth between the two original buses (usually with an amplifier) without considering the meaning of the signals.
  • Bridge: Like a repeater but it searches the signals and is more efficient. It looks at the destination address for each message and only fowrads it when its destined for a computer on the other side. Links two seperate LANs, useful for segmenting networks and extending them.
  • Switch: Essentially a bridge with multiple connections, allowing it to connect several buses rather than just two.
  • Router: Useful for connecting faraway networs, and they store data thats delivered in the form of packers. Different from the above becausew it provides links between networks while allowing them to keep their unique internal characteristics.
  • Gateway: A router that provides access to an external network to devices on a local network such as the Internet.
Client/server model: A popular convention for interprocess communication. A computer called a server runs a networked service such as a website which other computers called clients can connect to and use said service.
  • P2P:peer-to-peer: Unlike the client/server model, clients connect to each other and to provide and receive a service. Commonly used in video games and file sharing.
  • Distributed System: Software is ran across different computers connected together via a network.
  • Cloud Computing: A type of large scale distributed system which users can pay for a service to be ran on.
Internet Addressing
  • Tier 1: Large scale Internet Service Providers (ISP) that are the backbones of the Internet. They connect together tier 2 ISPs.
  • Tier 2: Medium scale ISPs that connect together tier 3 ISPs.
  • Tier 3: AKA Access ISPs, are the providers that bring internet access to consumers.
IP Adress
  • ICANN: "Internet Corporation for Assigned Names and Numbers". They are a nonprofit corporation that regulates Top Level Domains
  • DNS: "Domain Name System". A system that converts domain names into IP addresses.
  • TLD: "Top-level domain". Domains that are regulated by ICAAN which are the last section in a domain name such as com in google.com.
  • Country-code TLDs: Two letter TLDs used for specific countries (i.e. au for Australia and ca for Canada).
Internet Protocols
  • FTP: "File Transfer Protocol". Used to transfer files between computers.
  • HTTP: "Hyper Text Transfer Protocol". Used for almost every single website to transfer HTML and other files to users.
  • SMTP: "Simple Mail Transfer Protocol". Used to transfer emails between computers.
World Wide Web
  • URL: "Uniform Resource Locator". The unique address of each document on the web. There are four parts to it.
    1- The the protocol required to access the document
    2-Mnemonic name of host holding the document.
    3- Directory path indicating where the document is within the hosts file system.
    4- the document name.
  • Client-side activities: activities/actions, such as filling out a form, performed by a client (such as a browser).
  • Server-side activities: Activities, such as pulling up the results to a google search, performed by the server (such as a webserver).
Internet four-layer software: AKA TCP/IP Model
  • Application Layer: Consists of software units such as clients and servers that use Internet communications to carry out their tasks. This layer also consists of packages.It uses the transport layer to send and recieve messages over the internet. (Think of how you use a shipping company.)
  • Transport Layer: Accepts messages from the application layer and ensures that the messages are properly formatted for transmission over the internet. It divides the messages into small bits and sends them individually to make the transfer go smoothly.These segments are know as packets.
  • Network Layer: Determins where a packet has to go, and then gives the packet to the link layer for transmission. (Think of the machine that tapes labels to boxes being shipped.)
  • Link Layer: In charge of receiving and transmitting packets.It recieves it and then hands it up to the network layer to determine where it should go next.
Internet is:
  • Packet-switching network: A network type that sends data split across packets for faster and more reliable data transfer.
  • Fault tolerant: A system that can still operate during a fault such as hardware failure.
  • Redunadant: A system with extra resources in case of a failure in one of them such as server power supplies and disks arrays.
Cybersecurity
  • Malware: Abbreviation for "malicious software". One way for a computer system to be attacked via network connections.
  • Virus: Software that infects a computer by inserting itself into programs that already reside in the machine. When the host program is executed, so is the virus.
  • Worm: An autonomous program that transfers itself through a network, taking residence in computers and forwarding copies of itself to other computers. Can degarde applications and overload a network or internet.
  • Trojan horse: A program that enters a computer system disguised as a desireable program. Once in the computer, it can harm it either immediately or at a selected time.
  • Spyware: AKA Aniffing software. Software that collects information about activities at the computer it resides in, and sends the information back to the attacker.
  • Phishing: Obtaining information by asking for it while posing as a credibleand trustworthy group.
  • DoS attack: "Denial of Service". The process of overloading a computer with messages. This is done by planting software into numerous computers that will generate messages when a signal is given.
Protection
  • Firewall: A program to filter what traffic passes a point ina network.Can be installed and designed to block certain incoming/outcoming messages from certain addresses/origins. Can block DoS attacks.
  • Spam Filter: A variation of a firewall. It is a firewall designed to block unwanted mail.
  • Proxy Server: A software unit that acts as an intermiadiary between a client and a server. with the goal of shielding the client from adverse actions by the server.
  • Antivirus software: Used to detect and remove viruses and other infections. Defends against attacks via network connections. Antivirus software must be regularly maintained and updated.
  • Cryptography: The use of mathematical algorithms to obfuscate data in a way it can be reversed using only the correct key.
  • Public Key Encryption: Encryption systems designed to gaving knowledge on how messages are encrypted does not allow one to decrypt messages.
