Chapter 5 Programming Algorithms & Software Testing

Phases of computer development:

Phase 1
Understand the problem
Phase 2
Design an algorithm to solve the problem
Phase 3
Write the program code in appropriate language based on the algorithm
Phase 4
Enter the program and save
Phase 5
Execute the program
Phase 6
Test the program, find errors and debug, then retest
Phase 7
Document and evaluate the program

Programming Structures:

  1. Sequential structure: Top-down execution
  2. Decision Structures: if/then/else
  3. Iterative (Looping)
    • While Loop: Continue looping as long as something is true
    • Repeat/Until Loop: Continue looping until something is true
    • For Loop: Loop a given number of times
  4. Recursive Structures:
    • Repeat a set of instructions as a subtask of itself
    • Repetition Continues until: Degenerative case (AKA Base case), which is a termination condition, is met

Two Classic programming/database problems:

  1. Sorting:Descending order/Ascending order
    • Sorting numbers
    • Sorting strings (in accordance with ASCII values)
  2. Searching: find specific data that meet a certain criteria
    • In Database:
      • Filters: temporary search
      • Queries: search criteria are saved so they dont have to be recreated
    • Two Popular search techniques:
      • Sequential search: search from top to bottom or bottom to top, one item at a time
      • Binary search:
        • Only works if data is SORTED first
        • Start in the middle, then go half of that, etc.

Two Classic program goals:

Efficiency: Program takes minimal time and/or resources

Correctness: Program runs in expected ways and works for all expected conditions
